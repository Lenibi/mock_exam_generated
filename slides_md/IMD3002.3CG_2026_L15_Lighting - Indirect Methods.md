## Slide 10

Image -Based Lighting
´é¿Method Overview
´éñThe major benefit of IBL is that to setup a lighting 
environment that is relatively realistic is very quick and 
easy
´ü«Even if realism is not the goal, IBL can still be used to setup a 
relatively simple global lighting colour
´éñThe reason it is so simple is only an image of the light in 
an environment is required, and therefore any image 
capturing device (such as a camera) can be used; 
rendering is performed using relatively simple GI method (such as Final Gather) and therefore the render time is 
also small

## Slide 12

Image -Based Lighting
´é¿Capture the Light
´éñThe first step is to capture the light, in order to do that we 
need to measure the real- world illumination
´ü«This is often referred to as a ÔÇ£Light Probe ImageÔÇØ
´éñLight Probe Images are usually acquired using a photographic instrument (e.g. a camera), with specific

´ü«The image captured should be omni-directional, so that for

## Slide 15

Image -Based Lighting
´é¿Capture the Light: Linearly Proportional
´éñMost digital cameras capture images with pixel values 
that are not proportional to the scene light
´éñThe light levels are encoded non -linearly so that they 
appear more correctly (or pleasingly) on non -linear 
display devices (such as monitors or TVs)
´éñThe reason for this is because of the actual range allocated to light levels for typical RGB displays; this 
provides only a low dynamic range (the ratio between the 
darkest and brightest region)
´ü«Note: RGB is usually represented either by FF, 255 or 1.0f 
(depending on the system)

---

## Slide 16

Image -Based Lighting
´é¿Capture the Light: Linearly Proportional
´éñThe RGB limitation is due to the fact that each pixel value 
has 256 values for red, 256 values for blue, and 256 
values for green
´éñLooking at it a different way, the range between the 
darkest value and the lightest value is only 256
´éñAll light values have to fit into this range, and therefore depending on the exposure a lot of real light values have 
to fit into one pixel value
´éñIn this case, there is a need to use a higher number of bits 
per channel, such as 16 or 32

## Slide 18

Image -Based Lighting
´é¿Capture the Light: Linearly Proportional
´éñThe reason for using 16 or 32 -bits/channel, is that it 
provides a large range to store light information from 
dark to light
´éñWith 8 -bits per channel, there are only 28 (or 256) values

´éñWith 16 -bits/channel, 216 (or 65,536) values per channel

´éñWith 32 -bits/channel, 232 (or 4,294,967,296) values per 
channel are available
